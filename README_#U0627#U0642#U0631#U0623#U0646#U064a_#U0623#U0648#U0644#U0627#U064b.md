# تطبيق TAIO للأندرويد — تعليمات التشغيل

التطبيق ده مربوط بنفس مشروع Firebase بتاع الموقع (`taio-eg`)، يعني أي منتج أو أوردر
تضيفه من الموقع أو من التطبيق هيظهر في الاتنين فورًا، لأن الاتنين بيستخدموا نفس
الهيكلة بالظبط: `users/{uid}/store/{key}`.

## طريقة بناء الـ APK بدون تنزيل أي برنامج (الأسهل)

المشروع فيه ملف أتمتة جاهز (`.github/workflows/build-apk.yml`) بيخلي GitHub يبني
لك الـ APK أونلاين مجانًا. كل اللي محتاجه متصفح وحساب مجاني.

### الخطوة 1: اعمل حساب على GitHub
من https://github.com/signup (لو معندكش حساب بالفعل)

### الخطوة 2: أضف تطبيق أندرويد في Firebase Console
1. روح https://console.firebase.google.com وافتح مشروعك **taio-eg**.
2. من إعدادات المشروع (⚙️ Project settings) دوس **Add app** واختار أيقونة **Android**.
3. في **Android package name** اكتب بالظبط: `com.taio.app`
4. دوس **Register app** وحمّل ملف **google-services.json**.
5. حط الملف ده جوه فولدر `app/` في المشروع (نفس مكان `app/build.gradle.kts`)
   قبل ما ترفع المشروع على GitHub.

### الخطوة 3: ارفع المشروع على GitHub
1. من صفحة GitHub الرئيسية دوس **New repository**، سمّيه مثلاً `taio-android`،
   خليه Private لو عايز، وأنشئه.
2. في نفس الصفحة هتلاقي زرار **uploading an existing file** — دوس عليه وارفع
   كل ملفات المشروع (فك ضغط الملف اللي بعتهولك الأول وارفع المحتوى كله، بما فيه
   فولدر `.github`).
   > لو الرفع بالمتصفح رفض فولدرات فرعية، أسهل حل إنك تستخدم GitHub Desktop
   > (برنامج خفيف جدًا حوالي 100 ميجا بس، مش زي Android Studio) وتعمل منه Push
   > للمشروع كامل بضغطة زرار.

### الخطوة 4: خلي GitHub يبني الـ APK
1. بعد الرفع، روح لتبويب **Actions** فوق في صفحة الـ repo.
2. هتلاقي عملية اسمها **Build APK** شغالة أو خلصت (بتاخد 3-5 دقايق).
3. لو خلصت بعلامة صح خضرا ✅، دوس عليها، وتحت هتلاقي في قسم **Artifacts**
   ملف اسمه **TAIO-app-debug** — نزّله (هيجيلك كملف zip فيه الـ APK جواه).
4. فك الضغط وانقل ملف الـ APK لموبايلك وثبّته (لازم تفعّل "السماح بالتثبيت من
   مصادر غير معروفة" من إعدادات الموبايل أول مرة).

بكده الـ APK جاهز عندك من غير ما تنزل أي برنامج تقيل على جهازك خالص.

## الطريقة التقليدية (لو حبيت تستخدم Android Studio بدل كده)
- نزّل Android Studio من https://developer.android.com/studio (حوالي 1-1.5 جيجا)
- افتح المشروع منه، اعمل Sync، وبعدين
  **Build ← Build Bundle(s) / APK(s) ← Build APK(s)**

## مهم: قواعد الأمان في Firestore
تأكد إن الـ Firestore Rules بتاعة المشروع بتسمح للمستخدم بقراءة/كتابة بياناته هو
بس (نفس اللي شغالة مع الموقع أصلاً)، مثال:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/store/{docId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

## اللي التطبيق بيعمله دلوقتي (النسخة الأولى)
- تسجيل دخول / حساب جديد بنفس الإيميل والباسورد بتاع الموقع
- لوحة تحكم بإحصائيات سريعة
- المنتجات: إضافة / تعديل / حذف + حساب السعر تلقائيًا من التكلفة (زي الموقع بالظبط)
- الأوردرات: عمل أوردر جديد بسلة منتجات + قائمة كل الأوردرات + تغيير حالة الأوردر
- حاسبة تسعير + إعدادات نسبة الربح ومصاريف التغليف

## حاجات لسه ناقصة (ممكن نضيفها بعدين لو محتاجها)
- طباعة إيصال الشحن A4 (على الموبايل الأنسب PDF بدل طباعة مباشرة)
- أداة ختم الشعار على صور المنتجات
- تصدير Excel
- إرسال رسالة واتساب لتأكيد الأوردر

قولّي لو عايز أي حاجة من دول أو أي تعديل، وهكمل عليها.
